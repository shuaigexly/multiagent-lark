# 飞书知识库（Wiki）结构规范

## 知识库层级设计原则

### 最大层级：3层（超过3层导航困难）

```
一级（Space 下的一级节点）：业务域 or 职能域
  └── 二级：时间维度 or 功能模块
        └── 三级：具体文档
```

示例（产品团队）：
```
产品知识库
├── 产品规划
│   ├── 年度路线图
│   ├── Q3 规划
│   └── Q4 规划
├── 功能设计
│   ├── 核心功能 PRD
│   └── 迭代历史
├── 数据分析
│   ├── 月度数据报告
│   └── 用户研究报告
└── 会议纪要
    ├── 2025-Q1
    └── 2025-Q2
```

## 文档命名规范

**格式**：`[时间/版本] [动词/名词] [主题]`

| 类型 | 命名格式 | 示例 |
|------|---------|------|
| 报告类 | YYYY-MM [主题]报告 | 2025-01 用户增长分析报告 |
| 规划类 | QN YYYY [主题]规划 | Q1 2025 产品路线图 |
| 纪要类 | YYYY-MM-DD [主题]会议纪要 | 2025-01-15 增长策略决策纪要 |
| PRD类 | [功能名称] PRD vN.N | 消息通知中心 PRD v2.1 |
| 分析类 | [主题]分析 YYYY-MM | 竞品分析 2025-01 |

❌ 禁止使用：
- "新文档"、"未命名"、"复制的文档"
- "最新版"（无法区分真正最新）
- 无日期的报告类文档

## 索引页设计（每个二级节点建议有）

```markdown
# [模块名称] 文档索引

> 最后更新：YYYY-MM-DD | 维护人：XX

## 常用文档
- [重要文档1](链接) — 一句话说明用途
- [重要文档2](链接) — 一句话说明用途

## 按时间归档
| 时间 | 文档 | 状态 |
|------|------|------|
| 2025-Q1 | [文档名](链接) | 有效 |
| 2024-Q4 | [文档名](链接) | 归档 |

## 废弃文档
（移动到此处的文档，保留历史记录但不再维护）
```

## 文档生命周期管理

| 状态 | 标记方式 | 处理 |
|------|---------|------|
| 有效 | 无标记（正常） | 定期审阅 |
| 待更新 | 标题加 [待更新] | 90天内更新或归档 |
| 归档 | 移入"历史归档"节点 | 保留但不再维护 |
| 废弃 | 加 ⚠️ 废弃说明 + 替代文档链接 | 6个月后可删除 |

## 内容生成后的归档规则

每次 AI 生成分析报告后，建议：

1. **放置位置**：对应业务域 → 分析类 → 当月文件夹
2. **文档标题**：`[主题]分析 YYYY-MM-DD`
3. **添加标签**：AI生成、需人工核实
4. **通知方式**：@相关责任人 + 链接发群

## 从上下文推断归档位置

若 `feishu_context.drive` 中有文件，分析其路径规律：
- 识别现有的命名约定（是否有日期前缀、是否按部门分类）
- 建议新文档放在同一层级，遵循已有命名风格
- 若无规律，使用本规范默认结构建议

## 输出示例

当分析完成需要输出到知识库时：
```
建议将此报告保存至飞书知识库：
📁 路径：[业务域] > [分析类] > 2025-Q1
📄 文件名：增长漏斗分析 2025-01-15
🏷️ 标签：AI生成、增长分析、Q1-2025
👥 建议通知：@增长团队、@数据分析师
```
