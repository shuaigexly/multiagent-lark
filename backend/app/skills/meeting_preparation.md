# 会议准备技能

## 会议材料标准结构

```
# [会议名称] · 会前材料
**时间**：{日期} {时间}  **时长**：{N}分钟
**参会人**：{姓名列表}  **决策人**：{姓名}

---
## 会议目标
本次会议需要做出的决策（用动词开头）：
1. 决定：{具体决定内容}
2. 对齐：{具体对齐内容}

---
## 议题 {N}：{议题标题}（{预计时长}分钟）

**背景**（2-3句话）：{现状是什么}

**核心问题**：{本议题要回答的关键问题}

**已知信息**：
- {数据点1}
- {数据点2}

**待讨论选项**：
| 选项 | 优势 | 劣势 | 推荐度 |
|------|------|------|--------|
| A    |      |      | ⭐⭐⭐  |
| B    |      |      | ⭐⭐    |

**建议结论**：[主持人/提案人的推荐]

---
## 会后跟进清单
- [ ] [负责人] [截止日期] [行动项]
```

## SCQA 问题构建法
每个议题都用 SCQA 结构引入：
- **S**ituation（背景）：当前状态是什么，受众都知道的事实
- **C**omplication（冲突）：但有什么问题/变化出现了？
- **Q**uestion（问题）：因此我们需要回答什么问题？
- **A**nswer（答案）：我们的建议是什么？

## 决策准备要求
对于每个需要决策的议题：
1. 明确列出"不决策的代价"（时间窗口+损失）
2. 提供 2-3 个选项（不要只有一个选项）
3. 明确每个选项的不可逆程度

## 时间分配原则
- 背景说明：≤ 20% 时间（文字能看的不占会议时间）
- 讨论决策：≥ 60% 时间
- 行动确认：≥ 20% 时间
- 超时警报：议题超时立即标记，不压缩决策时间
