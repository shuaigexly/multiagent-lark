# 飞书通知路由规范

## 核心决策树：选择通知方式

```
分析结果需要通知 →
  是否需要立刻决策？
  ├── 是 → 直接发 DM + 电话/语音（卡片带行动按钮）
  └── 否 →
        影响范围？
        ├── 仅1-2人 → DM 私信
        ├── 小团队（≤8人）→ 发到对应业务群
        └── 全公司 → 发公告频道/全员群
```

## 通知方式对照表

| 场景 | 方式 | 格式 |
|------|------|------|
| P0风险触发（需立即决策） | DM + @手机 | 卡片（红色模板） |
| P1任务分配 | 目标群 + @负责人 | 卡片（含任务链接） |
| 分析报告完成 | 目标群 | 卡片（蓝色模板，附文档链接） |
| 周报/月报发布 | 管理层群 | 卡片 + 飞书文档链接 |
| 会议纪要 | 参会人群 + @所有参会人 | 文档链接 + 行动项摘要 |
| 数据异常预警 | 负责人 DM | 简短文本（带数字） |
| 一般信息同步 | 群消息 | 纯文本 |

## 紧急程度 × 通知规则

| 等级 | 触发条件 | 通知方式 | 响应期望 |
|------|---------|---------|---------|
| 🔴 极紧急 | P0风险、数据断崖、营收归零 | DM + @手机号 + 群@所有人 | 2小时内 |
| 🟡 紧急 | P1风险、KR严重落后 | DM 或 群@相关人 | 24小时内 |
| 🟢 普通 | 报告/任务/日历提醒 | 群消息（不@） | 本周内 |
| ⚪ 信息 | 数据更新/归档通知 | 频道订阅/文档通知 | 无期望 |

## @提及规则

### 何时 @someone
✅ 必须 @：
- 分配了任务（@负责人）
- 需要其决策（@决策人）
- 提到了其负责的数据出现问题（@数据负责人）

❌ 禁止 @：
- @所有人 发普通报告（除非是全公司 P0 事件）
- @老板 发每周例行数据（应由其团队主动查阅）
- 同一条消息 @5人以上（分拆发送或改发飞书文档）

### 飞书 @语法（用于生成消息内容）
```
@负责人 请于本周五前完成渠道 ROI 分析
<at user_id="ou_xxx">张明</at> 请于本周五前完成渠道 ROI 分析
```

## 消息长度控制

| 类型 | 长度限制 | 超出时 |
|------|---------|--------|
| 群消息文本 | ≤300字 | 改为文档链接 + 摘要 |
| DM 私信 | ≤500字 | 同上 |
| 卡片 content 字段 | ≤2000字 | 截断 + "查看完整报告：[链接]" |
| 行动项列表 | ≤8条 | 保留 P0/P1，其余放文档 |

## 通知消息模板

### 分析完成通知（群消息）
```
📊 [任务名称] 分析已完成

核心结论（1-2句）：
xxx

⚠️ 需关注：[最重要的1个风险/机会]

📄 完整报告：[飞书文档链接]
✅ 行动项（P0/P1，共N条）：
• [行动1]（@负责人，截止XX日）
• [行动2]（@负责人，截止XX日）
```

### P0/P1 风险预警（DM）
```
🔴 【紧急预警】[风险名称]

触发信号：[具体数字/事件]
影响范围：[预计影响]
建议决策：[A选项 or B选项]

需要您在 [时间] 前确认处理方向。
```

### 任务分配通知（DM）
```
Hi，分析结果中有 [N] 项任务需要您跟进：

1. [任务标题]（截止 [日期]）
   [1句背景说明]

2. [任务标题]（截止 [日期]）
   [1句背景说明]

如有疑问，报告链接：[链接]
```
