# 周报写作技能

## 周报标准结构

```
# [项目/产品名] 周报 · W{周次}（{日期区间}）

> 📌 **本周一句话**：[最重要的1个结论或风险，≤25字]

## 一、核心指标看板
[红绿灯表格 + 环比/同比数字]

## 二、本周进展（亮点）
[用"完成了什么 → 带来什么结果"格式]

## 三、风险与阻塞
[分级：🔴 紧急 / 🟡 关注 / 🟢 跟踪]

## 四、下周计划
[3-5条，每条带负责人和完成标准]

## 五、需要决策/支持
[明确列出需要管理层回答的问题]
```

## 核心指标看板格式
| 指标 | 目标 | 本周实际 | 上周实际 | 环比 | 状态 |
|------|------|---------|---------|------|------|
| DAU  | 10000 | 9800  | 9200   | ↑6.5% | 🟡 |
| 收入 | 50万  | 52万  | 48万   | ↑8.3% | 🟢 |
| 留存率| 40%  | 38%  | 41%   | ↓7.3% | 🔴 |

状态标准：
- 🟢 达成目标或超出 ≥5%
- 🟡 达成目标的85%-100%
- 🔴 低于目标85% 或 环比下降 >15%

## 异常解读必须回答的3个问题
1. **What**：具体哪个指标异常，幅度多大？
2. **Why**：根本原因是什么？（用归因优先级：外部→产品→运营→数据）
3. **So What**：对业务影响是什么？需要什么动作？

## 进展描述模板
❌ 差写法："完成了功能A的开发"
✅ 好写法："完成功能A上线（影响10%用户），A/B测试数据显示转化率+2.3%，预计本月带来新增收入8万"

## 风险分级标准
- 🔴 紧急：影响本周OKR达成，需立即介入
- 🟡 关注：如不处理，2周内影响下个里程碑
- 🟢 跟踪：已知风险，有缓解计划，暂不需要额外资源
