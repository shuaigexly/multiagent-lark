# 运营执行技能

## 任务优先级矩阵（四象限）
```
            紧急          不紧急
重要    【立即做】      【计划做】
        今天完成        排入日程

不重要  【委托他人】    【删掉/暂缓】
        找人做或外包     不要做
```

判断"重要"的标准：直接影响本期OKR关键结果

## OKR 对齐检查清单
在每个执行计划中确认：
- [ ] 此任务对应哪个 KR？（必须能对应，否则质疑其必要性）
- [ ] 完成标准是什么？（可量化的验收条件）
- [ ] 负责人是谁？（一人对一个任务，不允许"共同负责"）
- [ ] 依赖哪些外部输入？（列出依赖方和所需时间）
- [ ] 风险是什么？（最可能导致延期的1个原因）

## 跨团队协作风险识别
高风险信号：
- 任务跨越3个以上团队
- 关键路径上有唯一资源瓶颈（某人/某系统）
- 依赖外部团队但无明确承诺时间
- 有隐性依赖（A以为B在做，B以为A在做）

缓解措施：
1. 显式确认每个依赖的 Owner 和 Due Date
2. 关键节点提前3天提醒（不要等到截止日才催）
3. 高风险任务增加缓冲时间（建议 ×1.5）

## 执行追踪报告格式
每次追踪更新：
- **绿色**（按计划）：简短确认即可
- **黄色**（有风险）：说明风险 + 缓解方案 + 是否需要支持
- **红色**（已延期）：原因 + 新的完成日期 + 影响范围
