# 飞书多维表格（Bitable）设计规范

## 何时建议使用 Bitable

| 场景 | 推荐工具 | 原因 |
|------|---------|------|
| 多人协同追踪任务/项目 | ✅ Bitable | 结构化、可筛选、支持分配 |
| 定期更新的指标数据 | ✅ Bitable | 版本可追溯、支持图表 |
| 单次分析报告 | ❌ 用文档 | Bitable 的结构设计成本高 |
| 实时数据看板 | ❌ 连 BI 工具 | Bitable 不是实时数据库 |

## 表格字段设计原则

### 必有字段（每张表都需要）
- **主字段**：唯一标识（项目名/任务标题/指标名），文本类型
- **状态字段**：单选，颜色映射（绿/黄/红对应完成/进行中/阻塞）
- **负责人字段**：人员类型（绑定飞书用户）
- **截止日期字段**：日期类型（设置到期提醒）
- **更新时间字段**：自动填充（系统字段）

### 字段类型选择

| 数据类型 | 飞书字段类型 | 注意 |
|---------|------------|------|
| 文字描述 | 多行文本 | 支持 markdown |
| 数值/金额 | 数字（配合单位） | 用"进度"类型显示百分比 |
| 阶段/状态 | 单选 | 选项颜色要有含义 |
| 多属性标签 | 多选 | 不超过10个选项 |
| 关联其他表 | 关联字段 | 数据归一化必用 |
| 计算字段 | 公式 | 避免复杂嵌套 |

## 业务模板

### OKR 追踪表
```
字段：
- Objective（主字段，文本）
- Key Result（多行文本）
- 目标值（数字）
- 当前值（数字）
- 完成度（公式：当前值/目标值）
- 进度比（公式：完成度/应完成度）
- 状态（单选：🟢按计划/🟡需关注/🔴落后）
- 负责人（人员）
- 更新日期（日期）

视图：
- 默认视图（按Objective分组）
- 红灯视图（筛选状态=落后）
- 我的KR（筛选负责人=当前用户）
```

### 项目/需求追踪表
```
字段：
- 需求标题（主字段）
- 优先级（单选：P0/P1/P2/P3）
- 状态（单选：待确认/设计中/开发中/测试/已上线）
- 负责人（人员）
- 计划上线日期（日期）
- 实际上线日期（日期）
- 延期天数（公式：实际-计划）
- 关联 PRD 文档（URL）
- 备注（多行文本）

视图：
- 看板视图（按状态分组）
- 甘特图（按时间排期）
- 延期预警（筛选延期天数>0）
```

### 指标追踪表（周报数据源）
```
字段：
- 指标名称（主字段）
- 本周值（数字）
- 上周值（数字）
- 环比变化（公式：(本周-上周)/上周）
- 目标值（数字）
- 完成率（公式：本周值/目标值）
- 状态（公式：完成率>=1→达标；>=0.8→接近；否则→落后）
- 负责人（人员）
- 更新周（日期）
```

## 输出建议格式

当分析结果建议创建 Bitable 时，输出：

```
建议在飞书创建「[表名]」多维表格，用于追踪以下数据：

| 字段名 | 类型 | 用途 |
|--------|------|------|
| [字段1] | [类型] | [说明] |

推荐视图：
- [视图1]：[用途]
- [视图2]：[用途]

初始数据（来自本次分析）：
[表格数据]
```
