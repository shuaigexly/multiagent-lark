# 飞书会议与日历规范

## 会议类型与时长标准

| 会议类型 | 标准时长 | 参会人上限 | 产出物 |
|---------|---------|-----------|--------|
| 日常站会 | 15分钟 | 10人 | 阻塞项清单 |
| 周会 | 30-45分钟 | 15人 | 周报 + 下周计划 |
| 决策会 | 60分钟 | 6人 | 决策记录 + 行动项 |
| 复盘会 | 90分钟 | 8人 | 复盘报告 |
| 战略会 | 半天/全天 | 5-8人 | 战略文档 |

**黄金规则**：超过 8 人的会议通常是通知，不是决策。如果会议目的是"同步"，改成飞书文档。

## 会议议题结构（SCQA格式）

每个议题模板：
```
【议题名称】（≤15字，以动词开头）
时间：X分钟
背景（Situation）：当前状态是什么（1-2句）
冲突（Complication）：发生了什么变化/问题
核心问题（Question）：本议题需要回答什么
预期结论（Answer）：会议前的假设性结论（引导讨论，非定论）
决策类型：[A] 需现场决策 / [B] 需带方案讨论 / [C] 纯同步信息
```

## 参会人选择原则

**必须参会（M）**：对议题有决策权或直接责任
**应该参会（S）**：有专业输入贡献
**可选参会（O）**：仅需了解结论

生成参会人建议时，从 `feishu_context.calendar` 中识别：
- 近期同类会议的参会人作为基准
- 待办任务 `assigned_to` 中的相关责任人
- 避免邀请没有相关议题的高管（浪费其时间）

## 飞书日历事件输出格式

```json
{
  "summary": "Q3 增长策略决策会",
  "description": "议题：\n1. 渠道 ROI 对比（15分钟）\n2. 是否暂停 B 渠道（20分钟）\n3. Q3 增长目标调整（20分钟）\n\nPre-Read：[链接飞书文档]\n预期产出：渠道策略决策 + 调整后的 Q3 目标",
  "start_time": "2025-01-15T14:00:00+08:00",
  "end_time": "2025-01-15T15:00:00+08:00",
  "attendees_note": ["CEO", "数据分析师", "市场总监"],
  "location": "飞书视频会议 / 会议室XX"
}
```

## Pre-Read 文档结构（会前发出）

```
# [会议名称] Pre-Read
发送时间：会议前 24 小时

## 背景（2段以内）
## 需要做的决策清单
1. [决策点 + 选项A vs B + 影响]
2. ...

## 数据支撑（关键图表/数字）
## 请参会者提前思考的问题
```

## 会议纪要结构（会后30分钟内）

```
# [会议名称] 纪要
时间：YYYY-MM-DD HH:mm
参与者：XX、XX

## 决策记录
| 议题 | 决策结论 | 决策人 |
|-----|---------|--------|

## 行动项
| 事项 | 负责人 | 截止 |
|------|--------|------|

## 下次会议时间（如需）
```

## 从分析结论生成会议建议

当分析识别到以下信号时，建议召开对应会议：

| 信号 | 建议会议 | 优先级 |
|------|---------|--------|
| P0/P1风险触发 | 紧急决策会（24小时内） | 极高 |
| KR红灯且无缓解计划 | OKR对齐会（本周内） | 高 |
| 跨团队阻塞超过3天 | 协调解除会（本周内） | 高 |
| 季度数据出炉 | 经营复盘会（本月内） | 中 |
| 竞品重大动作 | 策略应对讨论会（2周内） | 中 |

## 禁忌

❌ 不要在描述/议题中写"其他事项"（无意义议题）
❌ 不要召开没有产出物定义的会议（说清楚会后应该有什么文档/决策/任务）
❌ 不要让会议时间超过标准时长 50%（重新设计议题，不要加时）
